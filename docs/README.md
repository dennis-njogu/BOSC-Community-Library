## 📝 Contributing to Documentation

### Found an issue?
- Open a GitHub Issue with:
  - What's unclear
  - What you expected
  - What would help

### Want to improve docs?
1. Create a branch: `docs/improvement-description`
2. Make your changes
3. Test that examples work
4. Submit PR
5. Include why the change helps

### Adding New Documentation

Before creating new docs:
1. Check if docs already exist elsewhere
2. Determine which section it belongs in
3. Follow existing format & style
4. Add reference to this README
5. Link from main navigation
